*** Análise da execução do fork() pelo programa "fork.c"

-- fork()
- Retorna -1, caso falhe
- Caso contrário, criará o processo filho e retornará o PID desse novo processo para o processo pai (armazenado em retval).
  O processo filho recebe 0 como retorno do fork (provavelmente para indicar que estamos no processo filho e não no processo pai).

*** Análise do valor da variável "x" na execução do programa "fork-print.c"

(Supondo que o fork() não falhe)

-- No processo pai:
- x = 0, printa 0
- fork() return PID do processo filho, então retval > 0 e, consequentemente, x continua sendo setado para 0
- printa 0 novamente

-- No proceso filho:
- x = 0, printa 0
- fork() retorna 0, então retval <= 0, o bloco do else é executado e, consequentemente, o x é incrementado em 1 unidade (x++).
- printa 1, já que o x foi incrementado

*** Análise do execve() no programa "fork-execve.c"

-- Quando o processo filho chama o execve(), ele para de executar o código do processo pai e passa a executar o código
